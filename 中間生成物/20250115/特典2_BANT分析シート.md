## BANTとは？

BANTは、見込み客を評価するための4つの基準です。

| 項目 | 意味 | 確認すること |
|------|------|-------------|
| **B**udget | 予算 | 購入する予算があるか？ |
| **A**uthority | 決裁権 | 決定権を持っているか？ |
| **N**eed | 必要性 | 本当に必要としているか？ |
| **T**imeline | 導入時期 | いつ導入したいか？ |

この4つが揃っている見込み客は、成約確度が高いです。

## 📋 各項目の評価基準

### Budget（予算）

| 記号 | 評価 | 説明 |
|------|------|------|
| ◎ | 確定 | 予算が確保されている |
| ○ | 見込み | 予算化の見込みがある |
| △ | 検討中 | 予算についてはまだ検討段階 |
| × | なし | 予算がない、または確認できていない |

**確認フレーズ**:
- 「ご予算はどのくらいをお考えですか？」
- 「他社と比較されていますか？予算感を教えていただけますか？」
- 「今期の予算に組み込まれていますか？」

---

### Authority（決裁権）

| 記号 | 評価 | 説明 |
|------|------|------|
| ◎ | 決裁者本人 | 目の前の人が決裁できる |
| ○ | 決裁者に近い | 決裁者に直接提案できる立場 |
| △ | 承認必要 | 上長の承認が必要 |
| × | 決裁権なし | 情報収集段階、決裁に関与しない |

**確認フレーズ**:
- 「最終的なご判断は、どなたがされますか？」
- 「今回のプロジェクトの責任者はどなたですか？」
- 「上長の方にもご同席いただくことは可能ですか？」

---

### Need（必要性）

| 記号 | 評価 | 説明 |
|------|------|------|
| ◎ | 明確 | 課題が明確で、解決したい意欲が高い |
| ○ | ある程度 | 課題は認識しているが、緊急性は低い |
| △ | 曖昧 | 課題がはっきりしていない |
| × | なし | 必要性を感じていない |

**確認フレーズ**:
- 「現在、どのような課題をお持ちですか？」
- 「この課題を解決しないと、どのような影響がありますか？」
- 「この問題は、いつ頃から感じていらっしゃいますか？」

---

### Timeline（導入時期）

| 記号 | 評価 | 説明 |
|------|------|------|
| ◎ | 今月中 | すぐにでも導入したい |
| ○ | 1-3ヶ月 | 近いうちに導入予定 |
| △ | 未定 | 導入時期は決まっていない |
| × | 当面なし | 今すぐの導入は考えていない |

**確認フレーズ**:
- 「いつ頃の導入をお考えですか？」
- 「導入のスケジュール感を教えていただけますか？」
- 「今期中に導入したいとお考えですか？」

## 🎯 BANTスコアと優先度

### スコア計算

◎ = 1点、○ = 0.5点、△ = 0.25点、× = 0点

| BANTスコア | 優先度 | アクション |
|-----------|--------|-----------|
| 3.5 - 4.0 | S | 最優先で対応。今週中にクロージング |
| 2.5 - 3.4 | A | 優先対応。1週間以内にフォロー |
| 1.5 - 2.4 | B | 通常対応。2週間以内にフォロー |
| 0.5 - 1.4 | C | 様子見。月1回のフォロー |
| 0 - 0.4 | D | 保留。ナーチャリング対象 |

## 📌 よくある失敗と対策

### 失敗1: 予算を聞けない
**対策**: 「他社ではこのくらいの予算感が多いですが、御社ではいかがですか？」と比較形式で聞く

### 失敗2: 決裁者が分からない
**対策**: 「最終的なご判断はどなたがされますか？」と直接聞く

### 失敗3: スコアが低い案件に時間をかけすぎる
**対策**: 週次でBANTスコアを見直し、S/A優先に集中する

### 失敗4: 更新を忘れる
**対策**: 商談後すぐに更新するルーティンを作る
